**Theme switching:** Detect system preference via `useColorScheme()` as the default. Allow the user to override via a value stored in AsyncStorage under `manas:settings:theme` (`'light'` | `'dark'` | `'system'`). The Settings screen should already handle this toggle — if it doesn't exist yet, add a theme selector there.

**Typography:** Use whatever font the rest of the app uses for UI chrome (labels, dates, stats, navigation). For the journal prompt text and entry body text only, load **Lora** (serif) via `expo-font` — this is the one typographic departure that makes the journal feel like a different space. The writing area should feel like paper, not an app.

**Header:** "Today" on the left, "Save" as a text button on the right (styled in `journalAccent` color). No back arrow visible while keyboard is up — handle with `KeyboardAvoidingView`.

**Body:**

2. **Text input** — large, multiline, no visible border, no placeholder text except a single thin blinking cursor. Font: Lora regular, textPrimary, 18px, lineHeight 28. This is the writing space — it should feel open and calm. `autoFocus: true`.

3. **Mood selector** — pinned above the keyboard using `KeyboardAvoidingView` + `position: absolute` bottom bar:

   - Text only, no emoji
   - Unselected: textMuted
   - Selected: mood color + a 2px underline in that color

4. **On Save:**

   - Pop back to JournalHomeScreen
