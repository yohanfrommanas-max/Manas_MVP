# Ghost Grid — Replit Implementation Guide (Complete)

---

## 1. What This Game Is

A spatial working memory game where the player observes a city grid populated with illustrated assets, watches them vanish, then reconstructs the layout from memory by tapping assets from a bank and placing them on a blank grid. Scoring is based on placement accuracy with partial credit for near-misses.

Core loop: **Observe → Ghost Transition → Reconstruct → Reveal → Result**

---

## 2. Grid Specification

**Fixed grid: 8 columns × 10 rows = 80 cells**

Grid dimensions never change between difficulties. The spatial field stays constant — only the number of assets changes.

**Cell sizing on a 390px wide screen:**
```javascript
const PADDING = 14; // each side
const GAP = 3;
const COLS = 8;
const cellSize = Math.floor((390 - PADDING*2 - GAP*(COLS-1)) / COLS);
// Result: ~43px. Cap at 40px max to leave room for asset bank.
```

**React Native implementation:** Use a plain `View` with `flexWrap: 'wrap'` and `flexDirection: 'row'`. Do NOT use FlatList — it introduces scroll behaviour that breaks the tap interaction. Render all 80 cells as `Pressable` components with fixed width, height, and margin of `GAP/2` each side.

**Grid line colour:** `rgba(255,255,255,0.05)` — very subtle. The grid reads as a spatial field, not a table.

## 6. Placement Generation

Runs at the start of each round. Places assets randomly without overlap.

```javascript
function generatePlacements(assetKeys) {
  const result = [];
  const usedCells = new Set();

  for (const key of assetKeys) {
    const def = ASSETS[key];
    let placed = false;
    let attempts = 0;

    while (!placed && attempts < 300) {
      attempts++;
      const col = Math.floor(Math.random() * (COLS - def.w));
      const row = Math.floor(Math.random() * (ROWS - def.h));
      let overlap = false;

      for (let r = row; r < row + def.h && !overlap; r++) {
        for (let c = col; c < col + def.w && !overlap; c++) {
          if (usedCells.has(`${c},${r}`)) overlap = true;
        }
      }

      if (!overlap) {
        result.push({ id: key, assetKey: key, col, row });
        for (let r = row; r < row + def.h; r++)
          for (let c = col; c < col + def.w; c++)
            usedCells.add(`${c},${r}`);
        placed = true;
      }
    }
    // If 300 attempts fail, skip this asset gracefully
  }
  return result;
}
```

## 7. Pixel Position Helper

Assets are absolutely positioned over the cell grid. This converts grid coordinates to pixel coordinates.

```javascript
function cellTopLeft(col, row, cellSize, gap) {
  return {
    x: col * (cellSize + gap),
    y: row * (cellSize + gap),
  };
}

function assetPixelSize(assetKey, cellSize, gap) {
  const def = ASSETS[assetKey];
  return {
    w: def.w * cellSize + (def.w - 1) * gap,
    h: def.h * cellSize + (def.h - 1) * gap,
  };
}
```

## 8. Collision Detection

Used during placement to check if an asset fits without overlapping existing assets.

```javascript
function canPlace(assetKey, col, row, excludeId = null) {
  const def = ASSETS[assetKey];

  // Bounds check
  if (col + def.w > COLS || row + def.h > ROWS) return false;
  if (col < 0 || row < 0) return false;

  // Collision check against existing player placements
  for (const p of playerPlacements) {
    if (p.id === excludeId) continue; // Skip self when repositioning
    const pDef = ASSETS[p.assetKey];

    const overlapX = col < p.col + pDef.w && col + def.w > p.col;
    const overlapY = row < p.row + pDef.h && row + def.h > p.row;

    if (overlapX && overlapY) return false;
  }

  return true;
}
```

The `excludeId` parameter is critical for the move mechanic — when repositioning a placed asset, you need to exclude it from collision detection against itself, otherwise it can never move.

## 9. Phase 1 — Observe

```javascript
function startObserve() {
  const cfg = DIFF_CONFIG[difficulty];
  const newPlacements = generatePlacements(cfg.assets);
  setPlacements(newPlacements);
  setPlayerPlacements([]);
  setSelectedBankId(null);
  setTimeLeft(cfg.observeTime);
  setTimerPct(100);
  setIsPaused(false);
  isPausedRef.current = false;
  setPhase('observe');

  const total = cfg.observeTime * 1000;
  let elapsed = 0;

  timerRef.current = setInterval(() => {
    if (isPausedRef.current) return; // Pause check inside interval

    elapsed += 100;
    const pct = Math.max(0, 100 - (elapsed / total) * 100);
    setTimerPct(pct);
    setTimeLeft(Math.ceil((total - elapsed) / 1000));

    if (elapsed >= total) {
      clearInterval(timerRef.current);
      startGhostTransition();
    }
  }, 100);
}
```

## 14. The Move Mechanic — Placing and Repositioning

This is the most important interaction in the game. Once an asset is placed, tapping it in the bank OR tapping it on the grid selects it again. Tapping a new grid cell moves it there.

```javascript
function handleCellTap(col, row) {
  if (!selectedBankId) return;

  const placement = placements.find(p => p.id === selectedBankId);
  if (!placement) return;

  const { assetKey } = placement;

  // Check if valid placement (excluding self from collision check)
  if (!canPlace(assetKey, col, row, selectedBankId)) {
    // Flash the cell red briefly
    triggerCellFlash(col, row, 'invalid');
    Haptics.notificationAsync(Haptics.NotificationFeedbackType.Error);
    return;
  }

  // Remove existing placement for this asset if it exists
  setPlayerPlacements(prev => {
    const without = prev.filter(p => p.id !== selectedBankId);
    return [...without, { id: selectedBankId, assetKey, col, row }];
  });

  Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium);
  setSelectedBankId(null); // Deselect after placing
}
```

**Tapping a placed asset on the grid to pick it up:**
```javascript
function handlePlacedAssetTap(id) {
  // Select it — next cell tap will move it
  setSelectedBankId(id);
  Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light);
}
```

**Visual states for placed assets:**
- Normal placed: coloured background, subtle border
- Currently selected for repositioning: brighter border, slight scale up (1.04), glow shadow
- Correct (reveal phase): sage green border and glow
- Wrong (reveal phase): rose red border, shake animation

## 17. Scoring Algorithm

```javascript
function submitReconstruction() {
  let total = 0;
  const breakdown = [];

  for (const correct of placements) {
    const player = playerPlacements.find(p => p.id === correct.id);
    let pts = 0;
    let status = 'missing';

    if (player) {
      // Chebyshev distance — max of column distance and row distance
      const dCol = Math.abs(player.col - correct.col);
      const dRow = Math.abs(player.row - correct.row);
      const dist = Math.max(dCol, dRow);

      if (dist === 0)      { pts = 100; status = 'exact'; }
      else if (dist <= 1)  { pts = 60;  status = 'close'; }
      else if (dist <= 2)  { pts = 30;  status = 'near';  }
      else                 { pts = 0;   status = 'wrong'; }
    }

    total += pts;
    breakdown.push({ id: correct.id, assetKey: correct.assetKey, pts, status, correct, player });
  }

  // Perfect placement bonus
  const perfect = breakdown.every(b => b.status === 'exact');
  if (perfect) total += 300;

  setRoundScore(total);
  setAllExact(perfect);
  setRoundBreakdown(breakdown);
  setSessionScore(prev => prev + total);
  setSessionRounds(prev => prev + 1);
  setPhase('reveal');
}
```

**Scoring table:**

| Distance | Points | Label |
|----------|--------|-------|
| 0 cells | 100 | Exact ✓ |
| 1 cell | 60 | Close |
| 2 cells | 30 | Near |
| 3+ cells | 0 | Wrong |
| All exact bonus | +300 | Perfect |

## 20. Asset Block Component

This is the most reused component in the game — renders both during observe (read-only) and reconstruct (interactive).

```jsx
function AssetBlock({ assetKey, col, row, cellSize, gap, isSelected, isPlaced, onPress, revealStatus }) {
  const def = ASSETS[assetKey];
  const pos = cellTopLeft(col, row, cellSize, gap);
  const size = assetPixelSize(assetKey, cellSize, gap);
  const C = useColors();

  const borderColor =
    revealStatus === 'exact'  ? C.sage :
    revealStatus === 'wrong'  ? C.error :
    revealStatus === 'close'  ? C.gold :
    isSelected                ? def.color :
    def.color + '80';

  const bgColor =
    revealStatus === 'exact'  ? C.sage + '20' :
    revealStatus === 'wrong'  ? C.error + '15' :
    def.color + '18';

  return (
    <Pressable
      onPress={onPress}
      style={{
        position: 'absolute',
        left: pos.x,
        top: pos.y,
        width: size.w,
        height: size.h,
        borderRadius: 8,
        backgroundColor: bgColor,
        borderWidth: isSelected ? 2 : 1.5,
        borderColor,
        alignItems: 'center',
        justifyContent: 'center',
        transform: [{ scale: isSelected ? 1.04 : 1 }],
        shadowColor: isSelected ? def.color : 'transparent',
        shadowOpacity: isSelected ? 0.4 : 0,
        shadowRadius: 8,
        elevation: isSelected ? 4 : 0,
      }}
    >
      <Text style={{ fontSize: cellSize * 0.5 }}>{def.emoji}</Text>
      <Text style={{ fontSize: 8, fontFamily: 'Inter_500Medium', color: def.color, letterSpacing: 0.5, marginTop: 2 }}>
        {def.label.toUpperCase()}
      </Text>
    </Pressable>
  );
}
```

## 24. What Not to Do

**Don't use drag-and-drop.** `react-native-gesture-handler` pan responders on a grid of 80 cells with multi-cell assets will be a debugging nightmare. The tap-to-select then tap-to-place mechanic is cleaner, faster on mobile, and builds in a fraction of the time. Players adapt to it immediately.

**Don't persist the observe grid between phases.** Regenerate the grid fresh in the reconstruct phase — don't try to animate the existing grid fading. Separate components per phase is cleaner architecture.

**Don't disable bank items after placing.** Keep them tappable so the player can pick an asset back up and move it. The "placed" state is visual only (checkmark, reduced opacity) — the asset is still selectable.

**Don't use FlatList for the cell grid.** Plain `View` with `flexWrap: 'wrap'`. FlatList virtualises rendering and will cause cells to disappear or reorder on scroll.

**Don't make the observe phase skippable.** No "I'm ready" button. The timer is the constraint — it is the cognitive load. Letting players skip it breaks the neuropsychological validity of the assessment.

**Don't clear `playerPlacements` when the user taps a bank item to reposition.** The asset should remain visually on the grid until they successfully place it somewhere new. The `selectedBankId` state is what marks it as "in hand" — the underlying placement data stays until overwritten.

## 27. Session Flow Summary

```
startObserve()
  → generatePlacements()
  → setPhase('observe')
  → start countdown timer
  → timer expires → startGhostTransition()

startGhostTransition()
  → setPhase('ghost')
  → 1.8s delay → startReconstruct()

startReconstruct()
  → setPhase('reconstruct')
  → player places assets
  → all placed → submit enabled

submitReconstruction()
  → score each asset
  → setPhase('reveal')
  → player reviews

Next Round button → startObserve() again
View Results button → setPhase('result')

Play Again → startObserve() with session reset
```
